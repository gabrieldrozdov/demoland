Making these tutorials. . . uh. . . fun?

I’m going to try to give you helpful examples with each tutorial, but learning how to code can still end up pretty dry. Here’s my idea to fix that:

  1. Each week, you’ll be asked to split into groups of 2 (or 3 if needed).
  2. Each group gets to pick a name. This can be a few words long.
  3. Throughout the lecture, I’ll ask you questions that you can buzz in to answer. If you get the question right, you get a point. (I’ll also need a scorekeeper!)
  4. At the end of the class, the winning team’s name gets added to a semester-long Mad Libs. You’re competing to see what this Mad Libs sentence ends up becoming.

For example, let’s say that a group named “Big dog” wins this week’s challenge. Next week, a group named “goes for a walk” wins, making the Mad Libs “Big dog goes for a walk.” We keep going until the semester is over.

Also, I’m going to embed a buzzer into each slide so that you can buzz in to answer questions.
