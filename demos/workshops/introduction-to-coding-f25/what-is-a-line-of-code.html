For many programming languages, line breaks are not essential (but they are super helpful). HTML doesn’t care about line breaks. Notice how the preview on the right doesn’t feature any line breaks, despite the editor having line breaks. Press the “WRAP” button on the bottom of the editor to make the code fit on the screen. HTML just cares about opening and closing tags. These tags indicate HTML elements, which contain different kinds of content.

Here is a paragraph! Notice how paragraphs break onto a new line of text. That’s because paragraphs are block-level elements.

Meanwhile, we use the strong element to bold text. That doesn’t break onto a new line of text. That’s because strong elements are inline elements.

Some HTML elements look different than others. This is a heading, specifically the Heading 1 element. There are 6 heading elements.

The div element is a generic block-level element, and the span element is the generic inline element.

HTML doesn’t even care if the opening and closing tags for an element are on the same line as its content.

Just make sure you don’t forget to put a closing tag for any opening tag!

If we wanted to make some ASCII art, which requires multiple spaces in a single line, we’d need to use the pre element for preformatted text.

You might be wondering, how is anyone supposed to know what HTML elements exist? There are two main sites people go to as references:

  • MDN, which is run by the browser company Mozilla and is basically the official reference for all things web-related.
  • W3Schools, which is just some random company that has become a semi-reliable web resource (and is usually easier than MDN to read).
