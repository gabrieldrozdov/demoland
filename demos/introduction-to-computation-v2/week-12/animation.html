The Animation Loop

A single renderer.render() draws one still frame. To animate, you need to call it repeatedly — once per frame, ~60 times per second.

requestAnimationFrame

The browser provides requestAnimationFrame (rAF) for this. It calls your function before the next screen repaint, perfectly in sync with the display's refresh rate.

// The loop calls itself recursively:
function animate() {
  requestAnimationFrame(animate); // schedule next frame

  // Update your scene here:
  cube.rotation.y += 0.01;

  // Draw the frame:
  renderer.render(scene, camera);
}

animate(); // kick it off once

Why not setInterval?

setInterval fires on a fixed clock, even when the tab is hidden or the GPU is busy. rAF pauses when the tab isn't visible (saving battery) and syncs to the actual display framerate.

The Clock

For smooth, framerate-independent movement, use THREE.Clock instead of counting frames. It gives you elapsed time in seconds.

const clock = new THREE.Clock();

function animate() {
  requestAnimationFrame(animate);
  const t = clock.getElapsedTime(); // seconds since start

  // Smooth sine wave regardless of framerate:
  cube.position.y = Math.sin(t * 2);

  renderer.render(scene, camera);
}
