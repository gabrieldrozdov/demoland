APIs (Application Programming Interfaces)

Note: this is a lot of info! You don’t need to understand it all right now. Instead, the big lesson is that objects are used to transmit and receive data all the time between websites.

APIs let us read and write data between computers and applications. On websites, APIs are often used to fetch data from online resources. For example, Wikipedia has an API that let’s us fetch a random assortment of articles. Other examples include Are.na’s API for getting content from channels, or the OpenWeather API for fetching forecasts.

To use Wikipedia’s API, we’d first want to read its documentation. This reveals that by inputting a URL in our web browser results in a JavaScript object, like in this example.

Let’s look at the following URL to see how it’s structured: https://en.wikipedia.org/w/api.php?​origin=*&action=query&​format=json&list=random&​rnnamespace=0&rnlimit=200

  1. The URL starts with the normal Wikipedia domain name with an extra w/ at the end: https://en.wikipedia.org/w/
  2. Then, we have api.php followed by a question mark ?
  3. Then, we get a series of URL parameters in the form parameter=value, with & signs in between these settings.

The actual settings for each of these parameters are things we’d have to determine from Wikipedia’s API documentation.

Accessing the URL reveals a JavaScript object (technically a JSON file) with a random assortment of article names. We can fetch this data in our code using the fetch function, which has a funny format (more info here) and eventually returns the object from the URL. We can then go through this object to get the specific data we want.
